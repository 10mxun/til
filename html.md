# HTML

<hr>

## HTML 이란
- 웹 페이지의 구조와 의미를 정의하는 하이퍼텍스트 마크업 언어이다.
- 태그(<>)를 사용하여 텍스트, 이미지 등의 콘텐츠를 브라우저에 표시하며, 웹사이트의 기본 뼈대를 만든다.
- 주로 CSS(스타일), JavaScript(기능)와 함께 웹 개발의 핵심 요소로서 사용된다.

<hr>

## HTML 기본 구조
- 문서 형식 선언(DOCTYPE), 루트 요소(html), 헤드(head) 본문(body)로 구성 된다.

<hr>

## 문서 형식 선언 (DOCTYPE)
- HTML 문서가 어떤 버전의 HTML이나 XHTML로 작성되었는지 웹 브라우저에게 알려주는 역할을 한다.
- 웹 브라우저에게 문서를 올바르게 해석하고 렌더링하는 데 필요한 정보를 제공한다.
- HTML5에서의 DOCTYPE 선언 방식 : <!DOCTYPE>

<hr>

## 루트 요소 (html)
- HTML 문서의 최상위 요소이며, 모든 다른 HTML 요소들을 포함하는 부모 요소이다.
- 모든 HTML 문서는 <html> 요소로 시작한다.
- HMTL5에서의 작성 방식 : <!DOCTYPE>
                        <html lang="ko">
                        </html>
<hr>

## 헤드 (head)
- HTML 문서의 메타데이터와 외부 리소스에 대한 정보를 포함하고 있는 부분이다.
- 화면에 직접적으로 보이진 않지만, 웹 브라우저가 문서를 처리하고 표시하는 데 중요한 역할을 한다.
- head 요소는 다음과 같은 요소를 포함한다.
  - 문서 제목 (title)
    - 문서의 제목을 지정한다. 브라우저의 제목 표시줄이나 북마크 목록에서 표시된다.
  - 메타데이터 (meta data)
    - 문서의 특정 정보를 정의하는데 사용된다. 주로 검색 엔진 최적화(SEO), 문자 인코딩, 뷰포트 설정 등을 지정한다.
    - 일반적으로 name과 content 속성을 사용하여 다양한 메타 데이터를 정의한다.
  - 스타일시트 및 외부 리소스
    - 문서에 스타일을 적용하기 위해 <style> 요소를 사용할 수 있으며, CSS 파일, JavaScript 파일 등과 같은 외부 리소스를 문서에 연결한다.

<hr>

## 본문 (body)

- HTML 문서의 실제 내용을 담고 있는 부분이다.
- 웹 페이즈를 구성하는 모든 텍스트, 이미지, 링크, 테이블, 폼 등의 콘텐츠를 포함한다.
- body 요소는 다음과 같은 요소를 포함한다.
  - 텍스트
    - 웹 페이지의 본문 내용을 작성할 수 있다. 제목, 단락, 목록 등의 텍스트 요소를 사용하여 웹 페이지의 구조를 정의할 수 있다.
  - 이미지
    - 웹 페이지에 이미지를 삽입할 수 있다.
  - 링크
    - 다른 웹 페이지로 연결하는 하이퍼링크를 만들 수 있다.
  - 위에 언급된 요소 외에 테이블 (table), 폼 (form)요소 등 다양한 구성요소가 포함된다.

<hr>

## HTML 주요 태그
- 제목
  - < h1 ~h6 >
- 문락
  - < p >
- 링크
  - < a >
- 이미지
  - < img >
- 목록
  - <ul/ol/li>
- 구조화
  - <div, span>

<hr>

## HTML 시맨틱 태그
- header : 웹 페이지나 특정 섹션의 도입부, 제목, 로고, 검색 폼, 내비게이션 메뉴 등을 포함하는 머리말 영역을 정의한다.
  - 사이트 로고, 제목, 검색 폼, 내비게이션 메뉴 등 핵심 정보를 페이지 상단에 배치하는데 사용된다.
  - 검색 엔진과 개발자에게 구조적 의미를 전달한다.
- footer : 웹 페이잔 섹션의 최하단 영역을 정의한다.
  - 저작권 정보, 회사 주소, 연락처, 개인정보 처리방침, 사이트맵 링크 등 웹사이트의 부가 정보나 마무리 내용을 담는데 사용된다.
- section : 문서의 주제별 구획을 나타낸다.
- article : 독립적인 콘텐츠 블록을 나타낸다.
- aside : 문서의 주요 내용과 간접적으로 관련된 콘텐츠를 나타낸다.
- main : 문서의 주요 내용을 나타낸다.
  - main 태그는 문서에 하나만 있어야 한다.
