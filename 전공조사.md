# Front end

<hr>

## 프론트엔드 (Front end)란

웹사이트나 앱에서 사용자가 직접 눈으로 보고 상호작용하는 영역(화면, UI/UX)을 개발하는 기술이다.

> - **사용자 접점** : 웹 브라우저를 통해 사용자가 마주하는 **화면 전체**를 뜻한다. <br>
> - **주요 역할** : 시각적 요소를 구성하고, 사용하기 쉬운 인터페이스(UI) 및 사용자 경험(UX)을 구현한다.<br>
> - **기술 스택** : HTML(구조), CSS(스타일), JavaScript(동작) 등으로 구성된다.

## 프론트 엔드 활용 예시

> - **웹 페이지 레이아웃** : 웹 사이트 접속 시 보이는 **메뉴바, 배너, 푸터**를 구성한다.<br>
> - **상호작용** : 버튼 클릭 시 동작, 폼 데이터 입력, 스크롤 반응, 팝업창 등이 있다.<br>
> - **데이터 시각화** : 그래프, 차트, 이미지 등을 화면에 표시한다.<br>
> - **API 연동** : 백엔드에서 받은 **데이터를 화면에 시각화하여 표현한다.**

## 프론트엔드와 혼용되는 용어

> - **클라이언트 사이드(Client-side)** : 사용자의 기기(브라우저)에서 실행된다는 의미에서 같은 영역을 지칭한다.<br>
> - **사용자 인터페이스 (UI) / 사용자 경험 (UX)** : 프론트엔드가 구현하는 **핵심 대상** <br>
> - **웹 퍼블리싱 (Web Publishing)** : 프론트엔드 업무 중 HTML/CSS를 이용해 **마크업하는 영역**을 뜻한다.

<hr>

## 프론트 엔드와 백엔드의 차이점

### 프론트엔드

> - **담당 영역** : 브라우저(Client)에서 **보이는 화면 (HTML, CSS, JS, React 등)** <br>
> - **핵심 목표** : 편리한 사용자 경험(UX)과 디자인을 구현한다.<br>
> - **주요 작업** : 웹/앱의 레이아웃, 버튼, 폼, 애니메이션, API 연동 등이 있다. <br>
> - **기술 스택** : HTML5, CSS3, JavaScript, React, Vue.js, Next.js 등이 있다. <br>

### 백엔드

> - **담당 영역** : **서버 및 데이터베이스 등을 설계하고 개발 및 운영**을 담당한다. <br>
> - **핵심 목표** : 데이터의 **논리적 처리와 시스템의 안정성 및 효율성**을 확보한다. <br>
> - **주요 작업** : 회원가입/로그인, DB 조회 및 저장, API 개발, 서버 배포 등이 있다. <br>
> - **기술 스택** : Java/Spring, Python/Django, Node.js/Express, PHP, SQL/NoSQL 등이 있다.

<hr>

## 프론트 엔드와 디자인의 차이점

| 구분      | 프론트 엔드                              | 디자인                             |
| --------- | ---------------------------------------- | ---------------------------------- |
| 주요 역할 | 디자인 구현, 기능 구현, 인터랙션         | 화면 기획, GUI(시각적 요소) 제작   |
| 핵심 도구 | HTML, CSS, JavaScript, React/Vue/Angular | Figma, Sketch, Photoshop, Adobe XD |
| 집중 분야 | 기능성, 성능, 웹 접근성, 브라우저 호환성 | 사용자 경험(UX), 시각적 미학(UI)   |
| 작업 형태 | 코드 작성, 실제 동작하는 웹페이지 빌드   | 프로토타입, 와이어프레임 생성      |

## 협업 구조

### 프론트 엔드 & 백엔드

<hr>

### 핵심 단계

> - **기획 / 설계 (초기)** : 어떤 데이터가 필요한지(요청, 응답 구조)를 정의한다. 이 단계에서 **API 명세서(API Contract)**를 작성하여 서로의 작업 기준을 세운다. <br>
> - **개발 (진행)** : 백엔드는 RESTful API나 GraphQL을 통해 데이터를 제공하고, 프론트엔드는 이 **API를 연결**하여 화면을 구성한다. <br>
> - **연동 / 테스트** : 실제 환경에서 프론트가 백엔드 서버를 통해 데이터를 조회/수정/삭제하는 기능을 확인한다.

<hr>

### 주요 협업 및 기술 구조

> - **API 명세 (API Contract)** : API URL 구조, 데이터 형태(JSON) 등을 미리 정의하여 **병렬 작업**을 유도한다. <br>
> - **BFF (Backend For Frontend) 패턴** : 프론트엔드의 **요구에 특화된 백엔드 레이어**를 구성하여 효율적인 데이터 처리 및 성능을 최적화한다. <br>
> - **JSON 기반 데이터 교환** : 프론트엔드와 백엔드 간의 데이터 교환을 위한 **표준 형식**이다. <br>
> - **버전 관리 (Git)** : 코드를 공유하고 통합하는 도구이다.

<hr>

### 성공적인 협업 팁

> - **소통 단절 방지** : 백엔드 개발자도 프론트엔드 작업의 UI 요소를 이해하고, 프론트엔드도 데이터 구조를 이해해야 한다. <br>
> - **빠른 연동 테스트** : 개발 초기에 **Mock API(가짜 데이터)**를 활용하여 백엔드가 완료되기 전에도 프론트가 개발을 진행할 수 있도록 한다.

<hr>

### 프론트엔드 & 디자인

> - **시스템 기반 협업** : 디자인 토큰(색상, 간격, 폰트)과 재사용 가능한 컴포넌트를 정의하여 **디자이너의 의도와 개발 구현 간의 불일치를 최소화**한다. <br>
> - **도구 및 언어 공유** : *Figma*를 주 소통 도구로 사용하며, 디자인 언어를 코드(CSS/JS)로 변환할 수 있는 환경을 구축한다. <br>
> - **기술적 설계 협업** : FSD(Feature Sliced Design) 같은 기능 중심 아키텍처를 도입하여 **도메인 기반**으로 폴더 구조를 나누고 독립성을 확보한다. <br>
> - **개발 과정의 커뮤니케이션** : 개발자는 디자인 레퍼런스(Codepen 등)를 참고하거나 컴포넌트 단위로 구현하여 중간 체크를 통해 **수정 비용을 절감**한다. <br>
> - **프로세스** : 기획/설계 → 디자인 → FE 개발 → 테스트 및 QA 순으로 진행된다.

<hr>

### 프론트엔드 & 클라우드

| 구분                     | 프론트엔드                              | 클라우드                                                        |
| ------------------------ | --------------------------------------- | --------------------------------------------------------------- |
| UI 설계 및 기능 정의     | 사용자 화면과 인터랙션을 설계한다.      | 해당 기능이 원활히 제공될 수 있도록 서비스 구조를 고려한다.     |
| 정적 파일 및 리소스 준비 | HTML, CSS, JS 등의 파일을 구성한다.     | 파일을 저장하고 전달할 수 있는 스토리지 및 CDN 환경을 준비한다. |
| 빌드 및 최적화           | 코드를 빌드하고 성능 최적화를 진행한다. | 효율적인 배포를 위해 캐싱 및 전송 구조를 설정한다.              |
| 배포 및 연결 설정        | 배포 가능한 결과물을 생성한다.          | 결과물을 서버에 배포하고 도메인 및 네트워크를 연결한다.         |
| 서비스 운영 및 성능 개선 | 사용자 경험을 개선하며 UI를 수정한다.   | 트래픽 관리와 모니터링을 통해 안정적인 서비스를 유지한다.       |

# Back end

<hr>

## 백엔드(Backend)란?

서버, 데이터, API 등 화면에서 **사용자가 볼 수 없는 뒷단**을 의미한다. <br>
보이지 않는 곳에서 **데이터와 로직을 처리하는 부분**이다. <br>

## 백엔드의 활용 예시

> - **SNS** : 로그인 정보 확인, 게시글 저장/불러오기, 좋아요 수 계산, 댓글 관리 등이 있다. <br>
> - **학교 시스템** : 출석 체크 기록, 성적 저장, 시간표 관리, 학사 정보 관리 등이 있다. <br>

<hr>

## 백엔드와 클라우드의 차이

### 백엔드

> - **역할** : 사용자의 요청 처리, 데이터베이스 연동, 인증/권한 관리 등을 한다.
> - **주요 기술** : Java Spring, Node.js, Python Django, SQL, REST API 등이 있다.
> - **목표** : 서비스의 **기능적 완성과 데이터의 무결성**을 확보하는 것이다.

### 클라우드

> - **역할** : 애플리케이션이 구동될 **서버의 가상화, 네트워크 설정, 스토리지, 자동 확장, 보안 설정** 등을 담당한다.
> - **주요 기술** : AWS(EC2, S3), Docker, Kubernetes, IaC(Terraform) 등이 있다.
> - **목표** : 애플리케이션을 안정적으로 **배포**하고 **확장성과 운영 효율성**을 확보하는 것이다.

<hr>

## 백엔드 & 디자인

### 핵심 협업 포인트 및 내용

| 구분                     | 백엔드                                         | 디자인                                                                         |
| ------------------------ | ---------------------------------------------- | ------------------------------------------------------------------------------ |
| 초기단계<br> (기획 / UI) | 데이터 구조를 설계한다.                        | 백엔드 개발자와 소통하며 구현 가능한 화면을 설계한다.                          |
| API 명세 공유            | 필요한 데이터를 **API 문서**로 전달한다.       | 화면에 어떤 정보가 어떻게 보여야 하는지 **(구성, 우선순위, 형태)**를 설계한다. |
| 실시간 소통과 피드백     | **기술적 제약사항**을 디자이너에게 피드백한다. | 협업 툴을 활용해 요구사항을 전달한다.                                          |
| 상호 이해                | 디자인 이해도가 높을수록 협업이 수월하다.      | 백엔드 이해도가 높을수록 협업이 수월하다.                                      |
| 사용자 중심 개발         | UX를 고려한 데이터 구조와 로직을 설계한다.     | 사용자 경험과 화면 구성을 설계한다.                                            |

<hr>

## 백엔드 & 클라우드

| 구분                 | 백엔드                               | 클라우드                              |
| -------------------- | ------------------------------------ | ------------------------------------- |
| 인프라 요구사항 정의 | 서버, DB, 트래픽 규모 등을 정의한다. | 이를 기반으로 인프라 환경을 설계한다. |
| 클라우드 환경 구축   | 실행 환경을 설정한다.                | 서버, 네트워크, 스토리지를 구성한다.  |
| 배포 및 실행         | 애플리케이션을 실행하고 테스트한다.  | 서버에 배포하고 네트워크를 관리한다.  |
| 성능 및 확장 관리    | 성능 개선을 진행한다.                | 오토 스케일링을 지원한다.             |
| 모니터링 및 유지보수 | 오류 수정 및 기능 개선을 수행한다.   | 서버 상태와 트래픽을 모니터링한다.    |

# 디자인

<hr>

## 디자인이란?

> - 사용자 경험(UX)과 사용자 인터페이스(UI)를 중심으로 화면과 흐름을 설계하는 과정이다. <br>

<hr>

## 디자인 & 클라우드

| 구분                  | 디자인                                               | 클라우드                             |
| --------------------- | ---------------------------------------------------- | ------------------------------------ |
| 요구사항 및 화면 설계 | 사용자 경험과 화면 구조(UI/UX)를 설계한다.           | 서비스가 운영될 환경을 고려한다.     |
| 디자인 시안 공유      | 완성된 화면 시안을 전달한다.                         | 인프라 요구사항을 검토한다.          |
| 클라우드 환경 구축    | 반응형 디자인을 고려한다.                            | 서버, 네트워크, 스토리지를 구성한다. |
| 배포 및 서비스 적용   | 배포된 화면이 **의도한 디자인과 동일한지** 확인한다. | 서비스를 서버에 배포한다.            |
| 운영 및 개선          | 사용자 피드백을 반영해 UX를 개선한다.                | 서비스의 안정성과 성능을 유지한다.   |

<hr>

## 클라우드

### 클라우드란?

> - 인터넷으로 연결된 원격 서버에서 데이터와 프로그램을 저장하고 처리하는 기술이다.

<hr>

### 핵심 특징 및 개념

> - 인터넷 기반
> - 가상화 기술
> - 비용 효율성
> - 확장성

<hr>

### 대표적인 사용 예시

> - 파일 저장 및 동기화
> - 스트리밍 서비스
> - 클라우드 오피스
> - 기업 인프라

<hr>

### 클라우드 관련 용어

> - 클라우드 컴퓨팅
> - 온라인 스토리지
> - 가상 서버
