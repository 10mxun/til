# JAVA Script

<hr>

## JAVA Script란

- HTML/CSS와 함께 웹페이지를 구성하는 핵심 프로그래밍 언어이다.
- 사용자 인터렉션, 애니메이션, 데이터 처리 등 웹사이트에 **동적인 기능(스크립트)을 부여**한다.
- **클라이언트 (브라우저)측에서 작동**하지만 Node.js를 통해 **서버측에서도 사용**할 수 있다.

<hr>

## 변수란

- 자바 스크립트에서 변수는 **데이터를 담는 그릇**을 의미한다.
- **할당 연산자(=)를 이용하여 값을 할당**시킨다.
- 식별자인 변수명을 사용하여 변수에 저장된 값을 참조한다.
- **한 줄에 변수 선언과 값 할당**을 할 수 있으며, **한번에 여러 변수**를 선언할 수 있다.

<hr>

## 변수 선언 방식

## var

- ES5에서 변수를 선언할 수 있는 **유일한 방법**으로 한 번 선언된 변수명에 **데이터를 재할당, 동일한 변수명으로 재선언**이 가능하다.
- 함수 레벨 스코프 : 함수 내부에서 선언된 함수만 지역변수로 한정하고, 함수 밖에서 선언한 var 변수는 전역 변수로 간주하여 **외부에서도 접근이 가능**하다. (이로 인한 문제가 발생할 수 있다.)
- 스코프 : *유효한 참조범위*의 함수 내에서 선언된 변수는 **함수 내에서만 유효하며 함수 외부에서는 참조할 수 없다**.
- 블록레벨 스코프 : *모든 코드 블록*내에서 선언된 변수는 **코드 블록 내에서만 유효하며 코드 블록 외부에서는 참조할 수 없다**.
- var 키워드 생략 가능 : var의 생략을 허용함으로써 **의도하지 않은 변수를 전역화** 시킨다.

<hr>

## let

- 한번 선언된 변수명에 데이터를 **재할당은 가능하나 동일한 변수명으로 재선언할 수 없다**.
- 동일한 변수명으로 재선언이 불가해 변수명이 동일하여 발생할 수 있는 문제를 방지할 수 있다.
- 변수 호이스팅이 발생하지 않음 : let 키워드로 선언된 **변수를 선언문 이전에 참조**하면 참조 에러가 발생한다.
- 블록레벨 스코프 : 모든 코드 블록내에서 선언된 변수는 코드 블록 내에서만 유효하며 코드 블록 외부에서는 참조할 수 없다.

<hr>

## const

- constant : **변하지 않는 값 선언** 시 사용한다.
- 변수를 선언하는 시점에 값을 할당하고, 값을 **재할당하거나 동일한 변수명으로 재선언 둘다 불가능**하다.
- let과 마찬가지로 **호이스팅이 발생하지 않으며, 블록 레벨 스코프를 지원**한다.

## 조건문이란

- **특정 조건을 만족할 때 실행하는 명령의 집합** 이며, 어떤 작업을 수행하고 싶을 때 사용하는 것이다. -특정 조건 충족시에만 특정 코드 블록을 실행하기 때문에 흐름을 효과적으로 제어할 수 있고, 프로그래밍적 유연성이 향상된다.

## 조건문의 종류

## if문

- 조건식을 만족일때만 코드를 실행하고 만족하지 않다면 else문이 실행된다.
- else문은 **선택사항**이며 작성하지 않을 경우 if문만 체크 후 다음 코드로 넘어간다.

## else if문

- 여러개의 조건을 검사할 때 사용할 수 있다.
- **횟수를 무한정** 늘릴 수 있으며 하나의 조건을 만족하면 코드를 실행하고 나머지 else if, else 조건문을 무시한다.
- else문은 선택사항으로 작성하지 않아도 된다.

## 삼항연산자

- 참/거짓 하나의 조건문만 사용할 때 삼항연산자를 사용하면 **한 줄로 표현**할 수 있다.
- 조건 : 평가되는 조건이다. 참이면 값1, 거짓이면 값2를 반환한다.
- 값1 : 조건이 참일 경우 반환된다.
- 값2 : 조건이 거짓일 경우 반환된다.

## 함수란

- 다른 객체처럼 속성 및 메소드를 가질 수 있어서 **일급 객체**이며 인자를 가질 수 있는 코드블록이다.
- 자체 범위를 가지고, **부모함수의 지역변수에 접근**할 수 있다.

## 함수를 사용하는 이유

- 코드를 **재사용**할 수 있다.
- 다른 인자를 사용하여 동일한 코드를 여러번 재사용하고 **다른 결과를 도출**할 수 있다.

## 함수의 특징

- 변수안에 담길 수 있다.
- 객체의 속성안에 메소드로 담길 수 있다.
- 다른 함수의 인자값을 전달될 수 있다.
- 함수의 **return값**으로도 사용할 수 있다.
- **배열의 값으로도 사용**할 수 있다.

## 함수 구문

- 함수 내에 선언된 변수는 함수의 지역변수가 된다. **함수 내에서만 접근**할 수 있고 함수가 시작될 때 생성되며 함수가 완료되면 삭제된다.
- 함수 이름에는 문자, 숫자, 밑줄 및 달러 기호가 포함될 수 있다.
- 괄호에는 쉼표로 구분된 매개 변수 이름이 포함될 수 있다.
- 함수가 실행할 코드는 { }안에 있다.

## 함수 선언

- function add (x, y) {
  var result = x + y;
  console.log(result);
  return result;
  }
  <br>
  => 함수 add를 선언하는 코드이다.

## 함수 이름 지정

- show.. : 무언가를 보여준다
- get.. : 값을 변환한다
- calc.. : 무언가를 계산한다.
- create.. : 무언가를 만든다.
- check.. : 무언가를 **확인하고 불리언등을 반환**한다.

## 즉시 실행 함수 (IIFE)

- 함수가 **선언되자마자 호출**되는 함수 표현식이다.

<hr>

## 매개변수

- function add (x, y) { <br>
  var result = x + y; <br>
  return result;<br>
  }<br>
- 첫번째 줄의 x, y를 매개변수라고 한다. **함수에 필요한 요소를 정의**하는 것이다.
- 매개변수의 정확한 값은 함수를 호출하는 순간에 결정된다.
- 함수를 호출할 때 **함수가 받는 값을 인자**라고 부른다.

<hr>

## Return 명령문

- return 명령문은 함수 실행을 종료하고, 주어진 값을 호출 지점으로 반환한다.
- 함수 내부에서 return 명령문이 실행될 경우, 해당 함수가 종료된다.
- return은 함수를 종료하는 기능과 더불어 어떠한 값을 반환할 수 있게 해준다.
- 함수 실행문이 있는 자리는 해당 함수의 return값으로 대체된다.

<hr>

## 화살표 함수

- ES6에서 도입된 function 키워드 대신 화살표를 이용하여 함수를 간결하게 정의하는 방식이다.
- 익명함수로만 사용 가능하며, 코드를 단축하고 특히 콜백 함수나 this 바인딩 문제를 해결할 때 매우 유용하다.

## 화살표 함수 / 일반 함수 차이점

- 화살표 함수와 일반 함수의 가장 큰 차이점은 **this 바인딩 방식과 문법 간결성**이다.
- 화살표 함수는 선언 시점의 상위 스코프 **this를 유지하고 생성자로 사용할 수 없다**.
- 일반 함수는 호출시점에 this가 결정되며 **생성자 함수로 사용 가능**하다.

## 자료구조

## 배열

- 0부터 시작하는 인덱스를 가진 순서가 있는 목록이다.
- 다양한 데이터 타입을 혼합하여 저장할 수 있는 동적 객체 구조이다.
- 크기가 고정되어있지 않아 **length 프로퍼티가 동적**으로 변하며, 대괄호로 생성하고 스택/큐 등의 자료구조를 쉽게 구현할 수 있다.

## 객체

## 객체란

- 여러 속성을 하나의 변수에 저장할 수 있도록 해주는 데이터 타입으로 **Key / Value Pair를 저장**할 수 있는 구조이다.

## 객체 구문

- var user = new Object();
  <br>var user = {};
  - 이름 : Minjun
    <br>나이 : 17
    - key : '이름','나이'
    - Value : 'Minjun',17

## 객체의 특징

- **객체는 변수**이다. 그러나 객체에는 많은 값이 포함될 수 있다.
- 객체는 중괄호를 표기하여 만들 수 있다.
- 객체는 각각의 key/value에 대한 정보를 나열할 수 있다.
- Key는 **문자열 또는 기호**여야 한다.
- Value는 **모든 유형**이 될 수 있다.
- 객체는 한 쌍의 key/value 뒤에 쉼표를 이용해 그 뒤에 오는 key/value와 구분해야한다.
- 객체에서 명명된 값을 Properties라고 한다.
- **변수는 예약어의 이름을 가질 수 없지만 객체는 어떠한 이름이어도 상관없다.**
- 객체 변수를 복사하면 참조가 복사되고 객체가 복제되지 않는다.

## 객체의 종류

- 배열
- 함수
- 객체
- 날짜
- 수학
- 정규 표현식
- **Boolean, 숫자, 문자열**은 객체가 될 수 있다. (new 키워드로 정의된 경우)
- JAVA Script에서 원시값을 제외한 모든 값은 개체이다.
  - 원시값 : 어떤 특성 또는 방법이 없는 값
  - 기본 데이터 형식 : 원시 값을 갖는 데이터

## 프로퍼티

- 객체의 속성을 나타내는 접근 가능한 이름과 활용 가능한 값을 가지는 특별한 형태이다.
- 특정객체가 가지고 있는 정보를 품고 있어 그 **객체가 가진 정보에 직접적으로 접근**할 수 있게 해준다.
- key : 빈 문자열을 포함한 모든 문자열 또는 symbol 값
- value : 모든 값

## 프로퍼티 추가 시 주의할 점

- 프로퍼티에 값을 할당하지 않을 경우
- delete 키워드를 사용하지 않고 프로퍼티를 삭제할 경우
- **프로퍼티 key로 예약어를 사용**하는 경우

## 메소드

- 객체가 가지고 있는 동작이며 **객체 내의 함수**를 칭한다.

## 메소드와 함수의 차이점

- 메소드는 객체가 가지고 있는 동작이기 때문에 메소드를 수행하기 위해서는 객체를 통해서 해당 메소드를 수행해야 하며 그 동작을 수행하는 주체는 객체이다.

## 객체 생성 방법

- 1. **객체 리터럴을 사용**하여 단일 객체를 정의하고 만든다.
- 2. **new 키워드를 사용**하여 단일 객체를 정의하고 만든다.

## 반복문

## 반복문이란

- 특정 코드를 여러번 실행할 수 있는 구조이다.
- 반복적인 작업을 자동화하고 코드의 가독성을 높이는데 유용하다.
- 대량의 데이터를 처리하거나, 동일한 작업을 반복해야할 때 매우 유용하다.

## 반복문의 종류

## for문

- 초기화, 조건, 증감 세가지 부분으로 구성된다.
- 장단점 : 명확한 구조로 이해하기 쉽지만, **복잡한 조건에서는 가독성이 떨어질 수 있다.**

## while문

- 조건이 **참인 동안** 코드를 실행한다.
- 장단점 : 조건에 따라 반복횟수가 달라질 때 유용하지만, 무한 루프에 빠질 수 있다.

## do ~while문

- 코드블록을 최소한 **한 번 실행 후 조건을 검사**한다
- 장단점 : 최소한 한 번은 실행해야하는 작업에서 유용하다.

## for ~in문

- 객체의 **열거 가능한 속성을 순회**시킨다.
- 장단점 : 객체 순회에 유용하지만, 배열에는 권장되지 않는다.

## for ~of문

- **반복 가능한 객체를 순회**한다.
- 장단점 : 반복 가능한 객체를 순회하는데 적합하다.

## forEach문

- 배열의 각 요소에 대해 **제공된 함수를 한번씩 실행**한다.
- 장단점 : 배열의 요소를 간단히 처리할 수 있지만, 중간에 반복을 멈출 수 없다.

## map 메서드

- 배열의 각 요소를 변환한 결과를 새 배열로 반환한다.
- 장단점 : 배열의 요소를 변환하여 새 배열을 만드는데 유용하다.

## forEach와 map의 차이점

- forEach는 배열의 각 요소에 대해 **함수를 실행하지만 반환값이 없다.** map은 배열의 각 요소를 변환한 새 배열을 반환한다.
